개인정보처리방침 (Privacy Policy)

최종 개정일자: 2026년 7월 20일

🛡️ 본 서비스는 "무지식 증명 (Zero-Knowledge) 아키텍처"를 채택하여, 환자의 어떠한 민감 의료 기록 및 개인정보도 서버에 영구적으로 보관하거나 저장하지 않습니다.

1. 개인정보의 처리 항목 및 목적

VitalPass QRDoc 서비스는 아래와 같은 최소한의 정보만 처리하며, 기기 외부의 상용 목적 등으로 데이터를 임의 수집/처리하지 않습니다.

  • 환자 기본 프로필 정보: 성함, 생년월일, 혈액형 (환자 본인의 신원 대조 목적)
  • 민감 의료 요약 정보: 보유 만성 질환 이력, 알레르기 유무, 비상 연락처 (응급 상황 시 조치 목적)
  • 복약 로그 기록: 의약품명, 복용량, 복용 빈도 및 기간 (의료진에게 복약 내역을 전달하는 목적)
  • 카메라 촬영 이미지: 처방전 스캔 목적 (Gemini AI API 분석 수행 후 메모리상에서 즉시 완전 삭제)

2. 개인정보의 안전한 로컬 보관 (스마트폰 저장)

본 서비스의 데이터 영구 보관은 오직 사용자의 스마트폰 내부에서만 이루어집니다.

  • 모든 건강 데이터는 모바일 운영체제의 최상위 물리 보안 영역(Keychain / Keystore)에서 발급된 임의의 256비트 보안 키로 암호화(AES-256)되어 저장됩니다.
  • 스마트폰 앱을 삭제하거나 데이터를 초기화하면 기기 내부의 보관 정보 및 암호화 키가 복구 불가능한 상태로 영구 삭제됩니다.

3. 종단간 암호화(E2EE) 및 제3자 제공 정책

의료진 공유를 위한 데이터 중계 시 제3자 유출을 원천적으로 차단합니다.

  • 의료진에게 QR 코드로 내보낼 때, 암호화된 파일만 임시 중계 서버에 등록되며, 데이터 복호화 키는 QR코드의 주소 해시값(# 뒤의 영역)으로 탑재됩니다. 인터넷 규격상 이 복호화 키는 중계 서버로 전송되지 않습니다.
  • 중계 서버에 등록된 암호화 파일은 환자가 설정한 만료 기간(기본 3분, 최대 30분)이 지나면 메모리에서 영구 삭제됩니다.
  • 조회 즉시 파기(Burn-After-Reading) 규정에 따라, 의료진이 웹 화면을 통해 1회 열람을 완료하는 즉시 서버에 잔존 데이터가 즉각 삭제됩니다.

4. 개인정보의 처리위탁 (AI OCR 분석)

사용자가 카메라를 이용하여 처방전을 스캔할 때 Google Gemini API 프록시 통신을 이용합니다. 전송 과정의 데이터는 오직 구조화 처리를 위해서만 일시적으로 사용되며, 학습용 데이터로 축적되지 않고 사용 목적 달성 즉시 물리적으로 파기됩니다.

5. 정보주체의 권리 및 행사방법

사용자는 본인의 데이터 주권을 가집니다. 앱 내 설정의 '데이터 초기화' 버튼 또는 애플리케이션 수동 삭제를 통해 기기 내 모든 정보를 즉시 소거할 수 있으며, 이 경우 회사의 서버에는 어떠한 백업본도 남아있지 않으므로 복구가 불가합니다.
